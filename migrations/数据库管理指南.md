## 🛠️ 工具概览

### Wrangler CLI（主要工具）

Wrangler CLI 是 Cloudflare 官方提供的命令行工具，用于管理 D1 数据库。

**主要功能**：
- ✅ 创建和管理数据库
- ✅ 应用和追踪迁移
- ✅ 执行 SQL 命令
- ✅ 本地和远程环境管理

**优势**：
- 官方支持，稳定可靠
- 自动记录迁移历史
- 与 Cloudflare 生态深度集成

### 辅助脚本（补充工具）

自定义脚本提供 Wrangler CLI 没有的便捷功能。

**保留的脚本**：
- `db-status.mjs` - 数据库状态检查
- `export-data.mjs` - 数据导出

**删除的脚本**：
- ~~`init-db.mjs`~~ - 功能与 Wrangler CLI 重复
- ~~`migrate-db.mjs`~~ - 功能与 Wrangler CLI 重复

## 📚 Wrangler CLI 命令

### 数据库管理

```bash
# 创建数据库
npm run db:create
# 或
npx wrangler d1 create spks

# 查看数据库信息
npm run db:info
# 或
npx wrangler d1 info spks

# 列出所有数据库
npm run db:list
# 或
npx wrangler d1 list
```

### 迁移管理

```bash
# 创建迁移文件
npm run db:migrate:create -- <migration_name>
# 示例
npm run db:migrate:create -- add_user_table

# 应用迁移（远程）
npm run db:migrate
# 或
npx wrangler d1 migrations apply spks --remote

# 应用迁移（本地）
npm run db:migrate:local
# 或
npx wrangler d1 migrations apply spks --local

# 列出迁移状态
npm run db:migrate:list
# 或
npx wrangler d1 migrations list spks
```

### SQL 执行

```bash
# 执行 SQL 命令（远程）
npm run db:execute -- --command "SELECT * FROM packages LIMIT 5"
# 或
npx wrangler d1 execute spks --remote --command "SELECT * FROM packages LIMIT 5"

# 执行 SQL 文件
npx wrangler d1 execute spks --remote --file=./path/to/file.sql

# 执行 SQL 命令（本地）
npm run db:execute:local -- --command "SELECT * FROM packages"
```

## 🔧 辅助脚本

### db-status.mjs - 数据库状态检查

**功能**：一次性查看完整数据库状态

**命令**：
```bash
# 查看完整状态
npm run db:status

# JSON 格式输出
npm run db:status:json

# 只查看表结构
npm run db:status -- --tables

# 只查看索引
npm run db:status -- --indexes

# 只查看统计
npm run db:status -- --stats

# 只查看迁移历史
npm run db:status -- --migrations
```

### export-data.mjs - 数据导出

**功能**：导出数据库数据为 SQL 或 JSON 格式

**命令**：
```bash
# 导出所有数据
npm run db:export

# 导出特定表
npm run db:export:packages
npm run db:export:arch

# 导出到文件
npm run db:export -- --output ./backup/data.sql

# 导出为 JSON 格式
npm run db:export -- --format json --output ./backup/data.json
```

## 🔄 常见工作流程

### 初始化新数据库

```bash
# 1. 创建数据库
npm run db:create

# 2. 应用初始迁移
npm run db:migrate

# 3. 验证数据库状态
npm run db:status
```

### 添加新迁移

```bash
# 1. 创建迁移文件
npm run db:migrate:create -- add_new_feature

# 2. 编辑迁移文件
# 编辑 migrations/0003_add_new_feature.sql

# 3. 本地测试
npm run db:migrate:local

# 4. 验证本地结果
npm run db:execute:local -- --command "SELECT * FROM new_table LIMIT 5"

# 5. 应用到远程
npm run db:migrate

# 6. 验证远程结果
npm run db:status
```

### 数据备份

```bash
# 1. 导出数据
npm run db:export -- --output ./backups/backup_$(date +%Y%m%d).sql

# 2. 验证导出文件
cat ./backups/backup_20260416.sql
```

### 查看数据库状态

```bash
# 快速查看
npm run db:status

# 详细查看（JSON）
npm run db:status:json | jq
```

## ✅ 最佳实践

### 1. 使用 Wrangler CLI 作为主要工具

```bash
# ✅ 推荐
npm run db:migrate
npm run db:migrate:list

# ❌ 不推荐
# 使用自定义脚本管理迁移
```

### 2. 本地测试后再应用到远程

```bash
# 1. 本地测试
npm run db:migrate:local

# 2. 验证结果
npm run db:execute:local -- --command "SELECT * FROM packages LIMIT 5"

# 3. 应用到远程
npm run db:migrate
```

### 3. 定期备份数据

```bash
# 每天备份
npm run db:export -- --output ./backups/daily_$(date +%Y%m%d).sql
```

### 4. 使用迁移文件管理数据库变更

```bash
# ✅ 推荐：创建迁移文件
npm run db:migrate:create -- add_column

# ❌ 不推荐：直接执行 SQL
npm run db:execute -- --command "ALTER TABLE packages ADD COLUMN new_field TEXT"
```
